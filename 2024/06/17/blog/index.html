安装 Hexo

安装教程

Hexo 是一个快速、简单且强大的博客框架，它允许你使用 Markdown（或其他渲染引擎）来编写文章，并且可以一键生成静态网页，详细信息请访问 Hexo 官网。

  1. 建立博客文件夹，例如我的文件夹建在 D:/coder/blog，后面以 . 代指博客根目录

  2. 使用 npm 安装 Hexo，在 ./ 路径下 右键 -> Git Bash Here，输入指令

  3. 安装插件

    • hexo 已默认安装的插件

    • 必须安装

    • 建议安装

    • 其它插件

  4. 本地预览

    在 Git Bash 中运行指令 hexo server 或 hexo s 来启动本地服务，然后在浏览器中访问 localhost:4000 即可，初始界面效果如下：

    \Typora\image\image-20240617231427338.png)

  5. 配置后台管理插件 hexo-admin

    启动本地服务后，浏览器访问 localhost:4000/admin 就可以在可视化界面中管理文章了。

    为了 hexo-admin 的管理安全性，需要添加访问权限认证，在右上角的 Settings 中，点击 Setup authentification here 之后，按文档步骤配置即可。

    重新启动本地服务，再访问 localhost:4000/admin 时，发现需要登录后访问，就是配置成功了！

    \Typora\image\image-20240617141613608.png)

  6. 为博客添加分类页面和标签页面

    用 Typora 打开 ./sources/categories/index.md，在它的头部添加 type 属性，

    同理，打开 ./sources/tags/index.md，添加 type 属性

  7. Hexo 常用命令归纳

    关于新建文章指令的参数，可参考官网 Hexo 写作

部署到 GitHub

部署到 github 教程

碎语

markdown 与 tags_plugins 的渲染有问题，吐槽一下，找了好久 bug，没有任何解决方案，只有加上这句碎语，才能使界面看起来美观些。

  1. 访问 GitHub 官网，按照指示注册 github 账号

  2. 新建 username.github.io 仓库

    注册成功后，在 github 首页点击 头像，打开 Your repositories，创建新的仓库，命名规则为 username.github.io

    科普：为什么要命名为 username.github.io

  3. 配置 git 与 github

    • 此处为全局配置，所以可以在任意位置打开 Git Bash，设置 GitHub 用户名和注册所用的邮箱地址

    • 检查是否配置成功

    • 上述配置完成后，为了能够在本地使用 git 管理 github 上的项目，需要绑定 SSHkey

    • 访问 github 账号，点击头像，打开 settings -> SSH and GPG keys，单击 New SSH key

      \Typora\image\image-20240617144700200.png)

    • Add SSH key 后，在 Git Bash 测试 sshkey 是否添加成功

  4. 配置 hexo 的部署插件

    • 确保你安装了插件 hexo-deployer-git

    • 打开 ./_config.yml，修改底部的 deploy 配置项。

      YAM 的缩进非常严格，注意缩进和空格

  5. 把本地 hexo 博客内容提交到 github 仓库

    部署完成后，在浏览器访问 https://username.github.io，就能访问自己的博客啦！

为避免升级主题后覆盖修改后的配置文件，请使用以下方法修改 Butterfly 的配置文件：

  1. 将 主题目录 的 _config.yml 备份至 hexo 的根目录，并重命名为 _config.butterfly.yml

  2. 所有的配置请在 _config.butterly.yml 文件中进行

    Hexo 会自动合并 主题目录 中的 _config.yml 和 hexo 根目录 的 _config.butterfly.yml，若存在同名配置，_config.butterfly.yml 配置文件的优先级高。
